# Individual/Group Project

**Deadline:** To be announced  
**Format:** Individual or small group project  
**Weight:** 10%

## Project Overview
This project will involve designing and implementing a relational database based on a real-world scenario or case study.

## Learning Objectives
By completing this project, you will:
- Apply database design principles learned in the course
- Create an Entity-Relationship (ER) diagram
- Normalize the database design
- Implement the database using SQL
- Write queries to manipulate and retrieve data
- Document your design decisions

## Project Components
1. **Requirements Analysis** - Identify entities and relationships
2. **ER Diagram** - Create conceptual design
3. **Schema Design** - Develop logical and physical design
4. **Implementation** - Create tables and relationships in PostgreSQL
5. **Sample Data** - Insert meaningful test data
6. **Queries** - Write SQL queries for key operations
7. **Documentation** - Explain your design and implementation

## Submission Requirements
- All files to be submitted via GitHub/Moodle
- Include ER diagram (image or tool output)
- SQL script file (.sql) with all DDL statements
- Documentation describing the design
